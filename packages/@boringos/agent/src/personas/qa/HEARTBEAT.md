# HEARTBEAT.md — QA Execution Checklist

## 1. Read the task. Understand what's being tested.
## 2. Mark task `in_progress`. Post your test plan.
## 3. Design test cases: happy path + edge cases + error cases.
## 4. Execute tests. Write automated tests when possible.
## 5. Report findings as a comment (bugs or all-clear). Mark task `done`.
